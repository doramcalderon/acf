\section{Ejemplo de uso}
	
		En esta sección se muestra un ejemplo de uso de la herramienta, desde que se registra el benchmark hasta la consulta de resultados, pasando por su ejecución.
		El ejemplo será de un benchmark en el que se compararán los algoritmos CLA y Direct Optimal Basis con un conjunto de implicaciones aleatorias.
		
			
			\begin{enumerate}
				\item Registrar un nuevo benchmark en la pestaña \textit{Nuevo} del área \textit{Benchmarks}.
					
						\begin{enumerate}
							\item Introducir el nombre en el campo \textit{Nombre}.
							\item Seleccionar la opción \textit{Aleatorio} del desplegable \textit{Seleccione Entrada}.\\
									
									\begin{figure}[h]
										\centering
										\includegraphics[scale=.75]{imagenes/figura58.png}
										\caption*{}
									\end{figure}
							\item Se abre el generador de implicaciones. Introducir el número de atributos, implicaciones y conjuntos a generar en sus correspondientes
									 campos y pulsar el botón \textit{Generar}.
									 
							\item Al finalizar la generación, se muestra un mensaje informando de ello. Aceptar el mensaje y cerrar el generador. Las rutas de las entradas
									 generadas se cargan en la lista de entradas.
									 
								\begin{figure}[H]
									\centering
										\includegraphics[width=.80\textwidth]{imagenes/figura59.png}
									\caption{Entadas insertadas desde el Generador}
									\label{fig:figura59}
								\end{figure}

									 
							\item Seleccionar los algoritmos CLA y Direct Optimal Basis de la lista de algoritmos con doble click. Éstos se añadirán a la lista de la
									derecha.
							\item Hacer click en \textit{Guardar}. Si todo ha ido bien, se mostrará un mensaje al pie informando de ello.
						\end{enumerate}
						
				\item Ejecutar el benchmark creado desde la pestaña \textit{Ejecutar}.
					
					\begin{enumerate}
						\item El benchmark registrado aparece en el árbol de benchmarks a la izquierda. Seleccionarlo haciendo click sobre él.
						
						\item La lista \textit{Ficheros de Entrada} se inicializa con las entradas definidas.
												
						\item El campo \textit{Salida}, se inicializa con la ruta \textit{[workspace\_dir]/[nombre\_benchmark]/output.}
							En este caso la ruta es un directorio, ya que se generarán \textit{n} salidas, una o más por algoritmo ejecutado.
							
					
						\item El modo de ejecución por defecto es \textit{Tiempos}. Seleccionar además el modo \textit{Estadísticas} para que calcule los tamaños y
							 cardinalidades de los sitemas implicacionales de salida.
						\item Pulsar el botón \textit{Run}. Aparece un indicador que estará visible durante la ejecución.
				
								\begin{figure}[H]
									\centering
										\includegraphics[width=.65\textwidth]{imagenes/figura63.png}
									\caption{Ejecución de un benchmark}
									\label{fig:figura63}
								\end{figure}

						\item Cuando finaliza la ejecución, el indicador desaparece y los resultados se muestran en la tabla de resultados.
						
								\begin{figure}[H]
									\centering
										\includegraphics[width=.75\textwidth]{imagenes/figura61.PNG}
									\caption{Resultados de la ejecución}
									\label{fig:figura61}
								\end{figure}

					\end{enumerate}
				
				
			\end{enumerate}

	\section{Implementación de Algoritmos}\label{sec:impl_algoritmos}
	
		Para que terceros puedan implementar algoritmos que puedan ser ejecutados por IS Bench, se ha creado una API con las
		interfaces y clases básicas para este fin que se incluye en el la librería \textit{is-algorithms}.
		
		El requisito mínimo para que un algoritmo pueda ser ejecutado desde IS Bench, es que implemente la interfaz 
		\texttt{ es.uma.pfc.is.algorithms.Algorithm} incluida en la librería \textbf{is-algorithms-1.0.0.jar}. 
		
		La dependencia a incluir en proyectos Maven es:
		
		\begin{verbnobox}[\small]
			<dependency>
	            <groupId>es.uma.pfc</groupId>
	            <artifactId>is-algorithms</artifactId>
	            <version>1.0.0</version>
	        </dependency>
		\end{verbnobox}
		
		La aplicación ejecuta el método \\
		\texttt{execute(input : ImplicationalSystem) : ImplicationalSystem} por lo que éste será en el que se implemente el
		algoritmo en cuestión.
		
		\begin{figure}[H]
			\centering
				\includegraphics[width=.65\textwidth]{imagenes/figura53.png}
			\caption{Interfaz Algorithm}
			\label{fig:figura53}
		\end{figure}

		
		La API proporciona una implementación abstracta de esta interfaz,\\ \texttt{es.uma.pfc.is.algorithms.GenericAlgorithm}, que
		implementa los \textit{getters} y \textit{setters} obligatorios, así como utilidades para la sustitución, adición y
		eliminación de implicaciones del sistema. El método\\  
		\texttt{execute(input : ImplicationalSystem) : ImplicationalSystem} no se implementa, ya que esto se deberá hacer en la
		implementación final.
		
		\begin{figure}[H]
			\centering
				\includegraphics[width=1.00\textwidth]{imagenes/figura54.png}
			\caption{Clase abstracta GenericAlgorithm}
			\label{fig:figura54}
		\end{figure}

		En el apartado \ref{subsec:clases_api_alg} se incluyen los detalles de esta API.
				
		A continuación se muestra un ejemplo de un algoritmo, que devuelve un sistema equivalente al de entrada cuyas conclusiones
		sólo contienen un atributo.
		
		\begin{verbnobox}[\small]
			public class UnaryAlgorithm extends GenericAlgorithm {
			    @Override
			    public ImplicationalSystem execute(ImplicationalSystem system) {
			        ImplicationalSystem outputSystem = new ImplicationalSystem(system);
			        outputSystem.makeUnary();
			        return outputSystem;
			    }
			}
		\end{verbnobox}
		
		Por lo que, para implementar un algoritmo y ejecutarlo desde IS Bench:
				
		\begin{enumerate}
			\item Crear un proyecto Java (JDK 1.8\_u65+) que tenga como dependencia la librería \textit{is-algorithms}.
			
			\item Crear una clase que implemente la interfaz \texttt{ es.uma.pfc.is.algorithms.Algorithm} o extienda de
					\texttt{es.uma.pfc.is.algorithms.GenericAlgorithm}.
					
			\item En el método \texttt{execute(input : ImplicationalSystem) : ImplicationalSystem} implementar el algoritmo en
				 cuestión.
				 
			\item Compilar el proyecto y generar el JAR correspondiente.
			
						
			\item Añadir la librería al workspace desde la pestaña \textit{Nuevo} del área \textit{Benchmarks}, con el botón
				 \includegraphics[scale=.65]{imagenes/botonadd.PNG}.
				 
				\begin{figure}[H]
					\centering
						\includegraphics[width=0.75\textwidth]{imagenes/figura64.png}
					\caption{Añadir librerías}
					\label{fig:figura_64}
				\end{figure}

			
			\item Incluir el algoritmo en un \textit{benchmark} como se explica en el apartado \ref{sec:registrar bench}, si no se
				 ha hecho ya en el punto anterior.
			
			\item Una vez incluido en el Benchmark, se podrá ejecutar el algoritmo bien individualmente o bien a través del
				 Benchmark desde la pestaña \textit{Benchmarks} / \textit{Ejecución}, como se explica en \ref{sec:ejecutar bench}.
		\end{enumerate}
