	\chapter{HERRAMIENTA PARA IMPLICACIONES}
	\newpage
	
	El PFC \textit{Algoritmos para la manipulación de implicaciones en ACF} tiene como objetivo la implementación de algoritmos
	relacionados con cierres y cálculo de bases y bases directas y realizar comparativas entre ellos.
	
	Por esto, surge la idea de desarrollar una herramienta que permita ejecutar los algoritmos implementados, medir su rendimiento
	y guardar los resultados y estadísticas de éstos permitiendo compararlos entre sí. A esta herramienta se le ha llamado IS Bench.
	
	Además, debido a que no existe un benchmark (banco de pruebas) al estilo de los que se utilizan en lógica para la comparación de 
	algoritmos,  se hace necesario un generador de conjuntos de implicaciones aleatorios que pueda servir de entrada a los algoritmos en cuestión.		
	
	\section{IS Bench}
		
		\textit{IS Bench} es una aplicación cuyo objetivo es ejecutar algoritmos sobre sistemas implicacionales, guardar sus
		resultados y compararlos posteriormente. Esta herramienta se pretende ofrecer a la comunidad FCA, para incorporarla en un futuro cercano
		a la librería de la Dra. K.Bertet para comparativas de algoritmos relacionados con las bases.
		
		La información se organiza en \textit{workspaces}, que no son más que directorios que contendrán los algoritmos y
		benchmarks registrados y ejecutados en él, así como los resultados (sistemas de salida, trazas y estadísticas) generados.
		
		Un \textit{benchmark} o \textit{experimento} es un conjunto de algoritmos que se ejecutan con uno o varios sistemas
		implicacionales de entrada comunes. 
		
		Un \textit{algoritmo} es una implementación Java que recibe como entrada un sistema implicacional y devuelve otro equivalente.
		Los algoritmos implementados inicialmente son algoritmos de cálculo de bases directas-optimales, pero la API no restringe su uso
		a este tipo de bases ya que la única condición es que su entrada sea un sistema implicacional al igual que la salida.
		
		
		El menú principal consta de las opciones:
		
		\begin{itemize}
			\item Algoritmos $\rightarrow$ Registro, para el registro de algoritmos.
			
			\item Preferencias $\rightarrow$ Workspaces, para la configuración de workspaces.
			
			\item Help con la ayuda de usuario.
		\end{itemize}
		
		El funcionamiento básico de la aplicación se resume en registro de algoritmos y benchmarks, ejecución de éstos y
		visualización de resultados. Por esto la aplicación se divide en tres zonas principales: \textit{Inicio},
		\textit{Benchmarks} y \textit{Resultados}.
		
		Con el área Inicio se pretende que el usuario pueda recordar las últimas acciones en el workspace actual al entrar en la
		aplicación.
		
		Se muestra un resumen de las últimas ejecuciones que contiene el nombre del benchmark, la fecha y hora de la última
		ejecución y un enlace para ejecutarlo de nuevo.

			guardan archivos relacionados para su uso en IS Bench.

			\begin{itemize}

				\item Archivos con sistemas implicacionales de entrada para los algoritmos a ejecutar. 
				\item Archivos con los sistemas implicacionales de salida de los algoritmos ejecutados. 
				\item Resultados.

		\subsection{Ejecutar algoritmos: Modos de ejecución}
		
			Recordemos que el objetivo de la ejecución de un algoritmo es, además de obtener un sistema implicacional de salida,
			obtener trazas y estadísticas que proporcionen información adicional para su comparativa.
			
			Para lo que se distinguen tres modos de ejecución:
			\begin{itemize}
				\item \textbf{Tiempos: }Se mide el tiempo de ejecución del algoritmo y se registra.
				
				\item \textbf{Con traza:} En este modo, se genera el archivo \textit{[nombre\_archivo\_salida]\_history.log}, que
				contiene la traza  de la ejecución.
				
				Esta traza será la que el desarrollador, en el momento de la implementación del algoritmo, considere que pueda ser
				de interés para el investigador.
				
				[nombre\_archivo\_salida] es el nombre base del archivo seleccionado para la salida del algoritmo. P.e., si el
				archivo que se ha tomado como salida es \textit{do\_output.txt}, el archivo de  traza será
				\textit{do\_output\_history.log}. 
				
				\item \textbf{Con Estadísticas: }En este modo, se genera el archivo \textit{[nombre\_archivo\_salida].csv}, en el
					que se guarda la evolución de los tamaños del sistema implicacional procesado. Esto se hace al final de la
					ejecución de cada algoritmo y se miden el tamaño y la cardinalidad del sistema obtenido.
					
					\textit{[nombre\_archivo\_salida]} es el nombre base del archivo seleccionado para la salida del algoritmo.
					Por ejemplo, si el archivo que se ha tomado como salida es \textit{do\_output.txt}, el archivo con las estadísticas
					será \textit{do\_output.csv}. 
					
					La información se guarda en archivos .csv para facilitar su visualización mediante tablas y gráficos. 
					
			\end{itemize}
			
			Estos modos no son excluyentes y pueden combinarse como el usuario considere necesario.
			
		\subsection{Ejecutar Benchmarks}\label{sec:ejecutar bench}
		
			La ejecución de un benchmark consiste en la ejecución secuencial del conjunto de algoritmos que lo componen con uno o varios sistemas implicacionales de entrada en común.
			
			Se genera un archivo de salida y varios adicionales según el modo de ejecución, por algoritmo ejecutado, además de un
			resumen de dicha ejecución con:
			
			\begin{itemize}
				\item La fecha y hora de ejecución
				\item Algoritmo ejecutado
				\item Sistema implicacional de entrada
				\item Sistema implicacional de salida
				\item Rutas de trazas y estadísticas generadas
			\end{itemize}
			
			Esta información se usará para la consulta de resultados.
			\newline
			
			La ejecución de benchmarks se hace desde la pestaña \textit{Ejecutar} del área \textit{Benchmarks}.
			
			\begin{figure}[H]
				\centering
					\includegraphics[width=.85\textwidth,frame]{imagenes/figura35.PNG}
				\caption{Pestaña Ejecutar Benchmark}
				\label{fig:figura35}
			\end{figure}

			Como se ve en la imagen, esta pestaña se compone de:
			
			\begin{itemize}
				\item Barra de herramientas
				\item Listado de Benchmarks
				\item Formulario para introducción de parámetros
				\item Visor de resultados
			\end{itemize}

			La barra de herramientas contiene tres botones \textit{Ejecutar}, que ejecuta un benchmark o algoritmo seleccionado, \textit{Parar} que para la ejecución en
			curso y \textit{Limpiar} que limpia tanto la selección en el árbol como el formulario.
			
			En la zona izquierda, se encuentra un árbol de dos niveles que contiene los benchmarks registrados en el workspace
			actual en el primer nivel, y los algoritmos que forman el benchmark en el segundo nivel. El contenido de éste árbol
			puede ser filtrado con el filtro situado en la parte superior de éste.
			
			El formulario de introducción de parámetros consta de:
			
		
			\begin{itemize}
				\item \textbf{Modo de ejecución:} Tres modos posibles \textit{Tiempos}, \textit{Traza} y \textit{Estadísticas}  no
					 excluyentes. Si se selecciona un benchmark en el árbol, el modo \textit{Traza} se deshabilita.
					 
					 Si se selecciona un algoritmo, los tres modos de ejecución están disponibles.
				\item \textbf{Ficheros de entrada:} Lista de los ficheros que contienen los sistemas implicacionales que servirán
					como entrada a los algoritmos a ejecutar. Para seleccionarlos, en el botón \textit{Seleccionar Entrada} se
					presentan dos opciones: \textit{Fichero} para seleccionar un fichero existente y \textit{Aleatorio} para
					generar un sistema implicacional aleatorio con el Generador de Implicaciones.
					
				\item \textbf{Salida:} Directorio en el que se guardarán los archivos resultantes de la ejecución. La ruta se puede
					introducir manualmente o seleccionándolo pulsando el botón
					\includegraphics[scale=.5]{imagenes/figura36.PNG}. La salida se puede generar en dos formatos: texto y prolog,
					que se pueden seleccionar en el desplegable que se encuentra a la izquierda del campo que contiene la ruta.
					El formato texto es la utilizada por defecto en la librería \textit{java-lattice}.
			\end{itemize}
			Los archivos generados tomarán nombres por defecto:
				\begin{itemize}
					\item de salida: [abreviatura\_algoritmo]\_[nombre\_archivo\_entrada].txt
					\item traza: [abreviatura\_algoritmo]\_trace.log
					\item estadísticas: [nombre\_algoritmo]\_[fecha\_hora\_ejecucion].csv, donde
						
								 [fecha\_hora\_ejecucion] es el momento de la ejecución con formato \\\texttt{yyyymmddhhMMss}.
				\end{itemize}
					
			Bajo el campo \textit{Salida}, se encuentra la tabla de resultados que contendrá los resultados de la ejecución actual agrupados por algoritmo. 			
			El contenido de cada una de las columnas es:
			
			\begin{itemize}
				\item \textbf{Nombre:} Sólo tiene valor en el primer nivel, y contiene el nombre del algoritmo.
				\item \textbf{Tiempo:} Tiempo de ejecución.
				\item \textbf{Tamaño:} Tamaño del sistema implicacional de salida.
				\item \textbf{Card.:} Cardinalidad del sistema implicacional de salida.
				\item \textbf{Entrada:} Nombre del archivo que contiene el sistema implicacional de entrada.
				\item \textbf{Entrada:} Nombre del archivo que contiene el sistema implicacional de salida.
			\end{itemize}
			
			Al hacer click con el botón derecho sobre alguno de los resultados, se muestra un menú contextual con las siguientes opciones:
			
			
			\begin{itemize}
				\item \textbf{Mostrar Log:} Muestra la traza generada si se ha ejecutado con el modo \textit{Tiempos} activo como se ve en la Figura \ref{fig:figura42}.
					También se podrá ver dicha traza haciendo doble click sobre el resultado.
				\item \textbf{Fichero de Estadísticas:} Abre el archivo de estadísticas generado si se ha ejecutado con el modo \textit{Estadísticas} activo.
			\end{itemize}

	\section{Generador de sistemas implicacionales aleatorios}
	
		El generador de sistemas implicacionales, permite a partir de un número de argumentos y un número de implicaciones, generar
		un sistema implicacional aleatorio. Se utiliza la librería \textit{java-lattices} de la doctora Karell Bertet, para el
		cálculo del sistema a partir de estos parámetros.
		
		Como se ve en la Figura \ref{fig:figura48}, este generador es una herramienta sencilla que consta de una pantalla principal
		dividida en tres zonas bien diferenciadas:
		
		\begin{itemize}
			\item Barra de herramientas
			\item Panel izquierdo en el que se introducen los parámetros para la creación del sistema: número de atributos, número de implicaciones,
					tipo, máximo y mínimo número de atributos en premisas y conclusiones y número de sistemas.
			\item Panel derecho que contiene el campo \textit{Salida} con la ruta absoluta del fichero en el que se guarda el
				 sistema implicacional generado, y un visor en el que se muestra dicho sistema.
		\end{itemize}

		\subsection{Generar un sistema implicacional}
			
			Para generar un sistema implicacional se deben seguir los siguientes pasos:
			
			\begin{enumerate}
				\item Introducir los parámetros para el sistema.
			
					\begin{itemize}
						\item \textbf{Atributos:} Número de atributos.
						\item \textbf{Implicaciones:} Número de implicaciones.
						\item \textbf{Tipo:} Tipo de los atributos. Se puede seleccionar entre tres tipos: númerico (1, 2, 3,...) ,
							alfabético (a, b, c, ...), alfanumérico (a1, a2, a3, ...).
						\item \textbf{Premisa: } Número mínimo y máximo de atributos en la premisa.
						\item \textbf{Conclusion: } Número mínimo y máximo de atributos en la conclusión.
						\item \textbf{Sistemas: } Número de sistemas implicacionales a generar. Por defecto, 1.
					\end{itemize}
					
				\item Pulsar el botón \textit{Generar}.
						Al pulsar este botón se genera el sistema con los parámetros introducidos y se muestra en el visor.
			
						\begin{figure}[H]
							\centering
								\includegraphics[width=0.85\textwidth]{imagenes/figura46.PNG}
							\caption{Generación de sistema aleatorio}
							\label{fig:figura46}
						\end{figure}
						
				\item Guardar el sistema en un fichero si se desea.\\
					  Para guardar el sistema, se debe introducir previamente la ruta absoluta del fichero en el que se desea
					  guardar.
					  Se puede hacer introduciéndolo directamente en el campo de texto, o buscándolo en el explorador, haciendo
					  click en el botón \textit{Examinar} (\includegraphics[scale=.5]{imagenes/figura36.png}) del campo
					  \textit{Salida}.
					  
					\begin{figure}[H]

		\subsection{Generar \textit{n} sistemas implicacionales}
		
			Como  se ha comentado anteriormente, es posible generar \textit{n} sistemas implicacionales con los mismos parámetros.
			Para ello, sólo se ha de introducir en el campo \textit{Sistemas} el número de conjuntos de implicaciones a generar.

			obligatorio que el campo \textit{Salida} contenga la ruta absoluta del fichero base, de otra forma, el botón 	
			\textit{Generar} no se habilitará.

	\section{Implementación de Algoritmos}
	
		Para que terceros puedan implementar algoritmos que puedan ser ejecutados por IS Bench, se ha creado una API con las
		interfaces y clases básicas para este fin que se incluye en el la librería \textit{is-algorithms}.
		
		El requisito mínimo para que un algoritmo pueda ser ejecutado desde IS Bench, es que implemente la interfaz 
		\texttt{ es.uma.pfc.is.algorithms.Algorithm} incluida en la librería \textbf{is-algorithms-1.0.0.jar}. 
		
		La dependencia a incluir en proyectos Maven es:
		
		\begin{verbnobox}[\small]
			<dependency>
	            <groupId>es.uma.pfc</groupId>
	            <artifactId>is-algorithms</artifactId>
	            <version>1.0.0</version>
	        </dependency>
		\end{verbnobox}
		
		La aplicación ejecuta el método \\
		\texttt{execute(input : ImplicationalSystem) : ImplicationalSystem} por lo que éste será en el que se implemente el
		algoritmo en cuestión.
		
		\begin{figure}[H]
			\centering
				\includegraphics[width=.65\textwidth]{imagenes/figura53.png}
			\caption{Interfaz Algorithm}
			\label{fig:figura53}
		\end{figure}

		
		La API proporciona una implementación abstracta de esta interfaz,\\ \texttt{es.uma.pfc.is.algorithms.GenericAlgorithm}, que
		implementa los \textit{getters} y \textit{setters} obligatorios, así como utilidades para la sustitución, adición y
		eliminación de implicaciones del sistema. El método\\  
		\texttt{execute(input : ImplicationalSystem) : ImplicationalSystem} no se implementa, ya que esto se deberá hacer en la
		implementación final.
		
		\begin{figure}[H]
			\centering
				\includegraphics[width=1.00\textwidth]{imagenes/figura54.png}
			\caption{Clase abstracta GenericAlgorithm}
			\label{fig:figura54}
		\end{figure}

		En el apartado \ref{subsec:clases_api_alg} se incluyen los detalles de esta API.
				
		A continuación se muestra un ejemplo de un algoritmo, que devuelve un sistema equivalente al de entrada cuyas conclusiones
		sólo contienen un atributo.
		
		\begin{verbnobox}[\small]
			public class UnaryAlgorithm extends GenericAlgorithm {
			    @Override
			    public ImplicationalSystem execute(ImplicationalSystem system) {
			        ImplicationalSystem outputSystem = new ImplicationalSystem(system);
			        outputSystem.makeUnary();
			        return outputSystem;
			    }
			}
		\end{verbnobox}
		
		Por lo que, para implementar un algoritmo y ejecutarlo desde IS Bench:
				
		\begin{enumerate}
			\item Crear un proyecto Java (JDK 1.8\_u65+) que tenga como dependencia la librería \textit{is-algorithms}.
			
			\item Crear una clase que implemente la interfaz \texttt{ es.uma.pfc.is.algorithms.Algorithm} o extienda de
					\texttt{es.uma.pfc.is.algorithms.GenericAlgorithm}.
					
			\item En el método \texttt{execute(input : ImplicationalSystem) : ImplicationalSystem} implementar el algoritmo en
				 cuestión.
				 
			\item Compilar el proyecto y generar el JAR correspondiente.
			
						
			\item Registrar el algoritmo en IS Bench, bien desde la ventana Nuevo Algoritmo (opción de menú \textit{Algoritmos}
					$\rightarrow$ \textit{Registro}) o bien desde la pestaña \textit{Benchmarks} / \textit{Nuevo}, como se
					explica en los apartados  \ref{sec:registrar algoritmos} y \ref{sec:registrar bench}.
			
			\item Incluir el algoritmo en un \textit{benchmark} como se explica en el apartado \ref{sec:registrar bench}, si no se
				 ha hecho ya en el punto anterior.
			
			\item Una vez incluido en el Benchmark, se podrá ejecutar el algoritmo bien individualmente o bien a través del
				 Benchmark desde la pestaña \textit{Benchmarks} / \textit{Ejecución}, como se explica en \ref{sec:ejecutar bench}.
		\end{enumerate}
